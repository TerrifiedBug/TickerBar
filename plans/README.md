## Findings considered and rejected (do not re-audit)

- **Crumb cached for process lifetime, no expiry**: self-heals — `invalidateAuth()` clears the crumb on any full-empty fetch result, so the next cycle re-auths. Only a minor proactive-expiry nicety.
- **Weak crumb content validation**: a `statusCode == 200` gate already exists; the crumb is an opaque TLS-delivered token. Over-validating risks breaking when Yahoo changes format.
- **Sparkle key via process-substitution `<(echo ...)`**: standard and safe (bash builtin, anonymous /dev/fd pipe). The proposed `mktemp` "fix" would regress by writing the key to disk.
- **Menu-bar NSImage re-render every body eval**: under `@Observable`, body re-evaluates only on real changes (rotation tick ~5s, price update); one tiny image alloc is negligible.
- **Sparkle `upToNextMajorVersion` floor of 2.5.0**: `Package.resolved` pins 2.8.1 exactly — range-in-manifest + exact-in-lockfile is the standard SPM convention.
- **Market-hours doc/heuristic reconciliation (docs)**: the heuristic is self-labeled "Approximate" with an upgrade note already recorded; only the lunch-break omission is non-obvious (covered by plan 010, not a separate docs plan).
- **Per-render perf micro-opts** (Calendar/TimeZone alloc in `isMarketOpen`; 3× portfolio reduce per body; currency-symbol dedup): real but imperceptible on a near-idle menu-bar app with a handful of symbols. The only perf item worth doing is the N+1 network fetch (plan 012). The currency-symbol DRY cleanup folds into plan 007.
